# A股量化交易系统 - 从模拟盘到实盘的分阶段上线计划

> 本文档定义了系统从当前状态到正式实盘交易的完整路线图。
> **核心原则：不跳过任何阶段，每一步都经过充分验证。**

---

## 阶段总览

```
回测验证 → 模拟盘运行 → 小资金实盘 → 正常资金实盘 → 多策略扩展
 (已完成)    (当前阶段)    (3个月后)     (6个月后)       (12个月后)
```

## 阶段一：模拟盘验证（当前 - 3个月）

### 目标
验证系统在真实市场环境下的完整运行能力

### 前置条件
- [x] 券商接口框架完成（TradeSimulator）
- [x] 双轨引擎完整实现
- [x] 风控模块可用
- [x] 学习器接入回测引擎
- [x] 实时行情支持分钟级数据

### 执行步骤

#### 1.1 系统测试（第1周）
```bash
# 运行完整测试
cd /opt/data/workspace/astock_quant
python live/test_paper_trading.py

# 预期结果：所有测试通过
```

#### 1.2 单标的模拟盘（第2-4周）
```bash
# 启动单标的模拟盘
python live/run.py --mode paper --symbol 000001 --strategy ma_cross

# 监控指标：
# - 每日交易次数是否符合预期
# - 止损止盈是否正常触发
# - 资金曲线是否合理
# - 日志是否有异常
```

**验收标准：**
- 连续运行 2 周无崩溃
- 所有订单正常成交或被合理拒绝
- 风控规则正常触发
- 日志完整无缺失

#### 1.3 双轨引擎模拟盘（第5-8周）
```bash
# 启动完整双轨引擎
python live/run.py --mode paper --dual-engine

# 监控指标：
# - 长线股策略优化是否正常
# - 波段股扫描是否有效
# - 网格做T是否执行
# - 学习器是否在更新参数
```

**验收标准：**
- 长线股和波段股同时运行正常
- 扫描选股有实际候选股票
- 学习器参数有更新
- 每日状态报告正常输出

#### 1.4 压力测试（第9-12周）
- **极端行情测试：** 模拟暴跌/暴涨场景
- **网络异常测试：** 模拟数据源断开
- **高频率交易测试：** 调整参数增加交易频率
- **内存泄漏检查：** 长时间运行监控内存

**验收标准：**
- 系统在各种异常情况下能正常恢复
- 无内存泄漏（连续运行 72 小时内存增长 < 10%）
- 数据源断开后能自动重连

## 阶段二：小资金实盘（模拟盘后 3 个月）

### 目标
用真实资金验证系统，但控制风险敞口

### 前置条件
- [ ] 阶段一所有验收标准达成
- [ ] 券商 API 对接完成（vnpy/CTP 或其他）
- [ ] 实盘环境搭建完成
- [ ] 人工干预机制就绪

### 执行步骤

#### 2.1 券商对接
```bash

pip install vnpy vnpy-ctp

# 配置券商信息（config.py）
BROKER_CONFIG = {
    "vnpy": {
        "gateway": "CTP",
        "user_id": "你的账号",
        "password": "你的密码",
        "broker_id": "经纪商代码",
        "td_address": "前置地址",
    },
}

# 测试连接
python live/test_broker_connection.py
```

#### 2.2 实盘环境准备
- **服务器：** 稳定的 Linux 服务器（推荐阿里云/腾讯云）
- **网络：** 低延迟网络，靠近交易所
- **监控：** 钉钉/企业微信告警
- **备份：** 自动备份配置和日志

#### 2.3 小资金运行（1-5万）

## 阶段三：正常资金实盘（小资金后 3 个月）

### 目标
逐步增加资金到正常水平

### 执行步骤

#### 3.1 资金逐步增加
- 第 1 月：5 万 → 10 万
- 第 2 月：10 万 → 20 万
- 第 3 月：20 万 → 目标资金

#### 3.2 持续监控
- 每日检查资金曲线
- 每周分析交易质量
- 每月评估策略表现

#### 3.3 风险控制
- 单票仓位不超过 20%
- 总仓位不超过 80%
- 最大回撤 15% 暂停交易
- 连续 3 日亏损减半仓位

## 应急计划

### 场景 1：系统崩溃
1. 自动停止所有交易
2. 发送告警通知
3. 记录崩溃日志
4. 人工检查后重启

### 场景 2：连续亏损
1. 触发最大回撤限制
2. 暂停所有新开仓
3. 分析亏损原因
4. 调整策略参数或停止策略

### 场景 3：券商异常
1. 尝试重新连接
2. 查询订单状态
3. 如有未成交订单，手动处理
4. 联系券商客服

### 场景 4：极端行情
1. 触发熔断机制
2. 暂停交易
3. 评估市场风险
4. 决定是否平仓
